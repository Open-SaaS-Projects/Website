Mobile Optimization Approaches

All include burger menu sidebar. Choose how to handle tables and layouts.

A

Card-Based Mobile Layout (Recommended)

Sidebar: Hidden by default, toggle with burger menu

Stats: Stack vertically (1 column) on mobile

Tables: Convert to card list on mobile - each job/application becomes a card with all info stacked vertically

Pros

  • Most readable on mobile - no cramped columns
  • Touch-friendly - larger tap targets
  • Progressive enhancement pattern
  • Common admin dashboard pattern

Cons

  • Requires more components/logic
  • Different UX between desktop/mobile
  • More vertical scrolling
B

Horizontal Scroll Tables

Sidebar: Hidden by default, toggle with burger menu

Stats: Horizontal scroll or 2-column grid on mobile

Tables: Keep table structure but allow horizontal scrolling with visible scroll hint

Pros

  • Minimal code changes - just wrap in scroll container
  • Consistent UX across all devices
  • Quick implementation

Cons

  • Horizontal scrolling is poor UX on mobile
  • Easy to miss columns off-screen
  • Actions may not be immediately visible
  • Feels like "desktop crammed into mobile"
C

Hybrid: Simplified Table + Drawer

Sidebar: Hidden by default, toggle with burger menu

Stats: 2-column grid on mobile (reduces to 1 column on very small screens)

Tables: Show only key columns (title + status), tap row to open drawer with full details and actions

Pros

  • Compact overview - see more rows at once
  • Progressive disclosure - details on demand
  • Good balance of table structure and mobile UX
  • ApplicationsPanel already uses this pattern

Cons

  • Requires drawer component for JobTable
  • Actions require extra tap
  • Moderate implementation complexity

My Recommendation: Option A (Card-Based)

Card-based layouts provide the best mobile experience for admin dashboards. They're touch-friendly, readable without zooming, and feel native to mobile. The extra implementation effort is worth it for usability. We can use Tailwind's responsive utilities (hidden/block classes) to show cards on mobile and tables on desktop.
